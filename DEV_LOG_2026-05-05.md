## 二、邀请制账号体系

### 2.1 数据库扩展
- `merchants` 表新增：`invite_code`、`invite_status`、`activated_at`、`invited_by`
- `influencers` 表同样扩展
- **存量数据默认 `invite_status='active'`**，不打断现有登录

### 2.2 后端接口
- 新增 `routes/invitations.js`：
  - `GET /validate?code=xxx` 校验邀请码，返回脱敏身份信息
  - `POST /activate` 激活账号（验证身份 + 设置密码）
- 改造 `POST /merchants` 和 `POST /influencers/add`：支持 `invite_mode=1` 走邀请制
- 新增 `GET /merchants/:id/invite-code`（重取邀请链接）
- 新增 `PUT /merchants/:id/invite-status`（启用/停用）
- 达人侧同样一套接口：`invite-code` / `invite-status` / `reset-password`（之前缺的，一并补齐）
- 登录时校验 `invite_status`：`pending` / `disabled` 禁止登录

### 2.3 手机号登录
- 商家登录优先匹配手机号，name/company 作为兼容回退
- 激活成功后明确告知"登录账号（手机号）：138xxxx"
- 激活后自动跳转到登录页并预填手机号（`?u=` 参数）
- 登录页 placeholder 更新："手机号 / 视频号账号 / 管理员账号"

### 2.4 前端改造
- 激活页（`public/index.html` 新增 `#activate-page`，URL `?activate=xxx` 触发）
  - 身份验证：商家输完整公司名、达人输完整视频号账号名
  - 密码最低 6 位 + 两次输入一致校验
- 商家管理
  - "+ 添加商家" → "+ 邀请商家"
  - 新增状态列（待激活/已激活/已停用/已删除四种徽标）
  - 按状态动态显示操作按钮
  - 邀请成功后弹出模态框：完整链接 + 登录账号提示 + 一键复制
- 达人管理
  - "+ 添加达人" → "+ 邀请达人"
  - 表格同步加状态列、操作按钮按状态分支
  - 销售普管菜单加入达人管理入口，自动按 sales_owner_id 过滤

### 2.5 销售普管邀请自动归属
- 销售邀请商家/达人时 `sales_owner_id` 前端强制为 `currentUser.id`
- 表单中隐藏"归属销售"选择器，显示"邀请后将自动归属到你（xxx）名下"提示

### 2.6 商家批量导入
- 新增 `GET /merchants/excel/template` 下载导入模板（含填写说明 sheet）
- 新增 `POST /merchants/excel/import`：
  - 事务导入，单条失败不影响全局
  - 去重：phone/company 任一已存在则跳过
  - **全部走邀请制**（每条自动生成 invite_code，status=pending）
  - 销售归属：按 Excel 姓名自动匹配管理员；销售普管上传时默认归属自己
- 前端加"批量导入"按钮 + 上传区 + 结果展示（成功/跳过/失败分色 + 前 20 条异常明细）

## 三、修复的其他问题

1. **撮合管理看板卡 source_type 徽标对比度不足**：加边框 + 圆角 + 加粗，灰/蓝/紫三色区分清晰
2. **tooltip 被列头遮挡**：CSS 规范的 containing block 陷阱——`.kanban-column:hover` 的 transform 让 fixed 失效。解法是 mouseover 时把 tooltip `appendChild` 到 `document.body`，物理脱离 DOM 链
3. **商家历史 demo 数据 source_type 全是"手动创建"**：按 source 文本回填 12 条"邀约转化" + 12 条"申请转化"，让色彩样本丰富
4. **统一术语**：所有"⭐ 升级佣金" / "五角星" 改为"⬆ 升级" / "升级标签"

## 六、架构状态一览

### 后端路由（routes/）
```
admins.js          管理员管理
cooperation.js     合作邀约 + 撮合工单（含单向孵化）
demands.js         商家货盘（图书/课程）+ 达人需求
excel-import.js    通用 Excel 导入
influencers.js     达人管理（含邀请/启停/重置密码/excel模板）
invitations.js     ✨ 新增：邀请码校验 + 激活
merchants.js       商家管理（含邀请/启停/重置密码/excel模板/批量导入）
recruitments.js    商家招募
stats.js           数据统计
```

### 数据库关键字段
```
merchants:
  invite_code, invite_status (pending/active/disabled), activated_at, invited_by
influencers:
  invite_code, invite_status, activated_at, invited_by
matchmaking:
  source_type (手动创建/邀约转化/申请转化), cooperation_id
```

### 登录逻辑
```
POST /api/login (username, password)
  1. admin 精确匹配 username
  2. merchant: phone 优先 → name/company 兜底
     若 invite_status != active 则禁登
  3. influencer: video_account_name 匹配
     若 invite_status != active 则禁登
```
